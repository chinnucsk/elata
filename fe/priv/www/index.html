    The application intent is the behavior analysis of cloud-base software components using live telemetry gathered as the cloud sustain production load. The objectives are control of grade of service is fulfilling the requirements; quality assessment of distributed software architecture; resolution short term capacity decisions include for example the determination of optimal configuration, the number of lives servers; resolution of long term capacity decisions such as decisions concerning the development and extension of data- and service architecture, choice of technology, etc; to provide granular latency control for diverse data and computational load in hybrid Cloud architectures.

    "With great power comes great responsibility". Users of the service are empowered with ability to define latency tracking use-cases that are executed agains live systems in 24/7 manner. Please consider carefully each use-case defined by you. Please pay speciall attention when use-case covers 3rd party services.

    User credential and e-mail is required for audit purposes only. User profiles are public.

  • Note

    You are about to create a observer process against live system that might cause 24/7 load to live system. Please consider carefully necessity of the use-case defined by you.

    Uri defines behavior of observer process and specified system end-point. The following schemas are supported:

    dns://myhost.com - measures a time required to resolve IP address of myhost.com

    tcp://myhost.com - measures a time required to establish TCP/IP connection (default port is 80).

    ssl://myhost.com - measures a time required to establish SSL connection (default port is 443).

    http://myhost.com/mydoc.html - measures a time required to fetch mydoc.html from myhost.com over HTTP (defat port is 80).

    https://myhost.com/mydoc.html - measures a time required to fetch mydoc.html from myhost.com over HTTPS (defat port is 443).

    Foreach use-case the tool aggregates, reports and visualize various KPIs such as dns resolution time; time to establish tcp/ip connection, time to establish ssl connection, time to first byte, time to meaningful response (time to receive body), size of response, average size of ip packet, number of ip packets.
